# M3 视觉规范：温暖赛博屋顶

状态：M3 制作基线  
更新日期：2026-07-18

本规范定义首版的视觉语言、像素系统、响应式构图和素材验收标准。目标不是堆叠赛博朋克符号，而是让玩家在一眼看懂落点的前提下，感受到温暖、可爱、略带冒险感的夜间城市。

## 1. 创意北极星

一句话方向：**一只带着小围巾的奶油橘猫，在仍有人生活的温暖未来城市屋顶上冒险。**

必须同时成立：

- 温暖：窗灯、砖墙、蒸汽和生活痕迹带来安全感。
- 可爱：猫的圆润轮廓和动作有弹性，但不幼儿化。
- 清晰：角色、可落平台、空隙和危险物在一秒内可区分。
- 未来感：使用克制的霓虹、电子招牌和远景交通，不依赖冷硬机械堆砌。

避免阴冷反乌托邦、满屏高饱和霓虹、写实暴力、过度黑暗，以及直接联想到现有商业游戏或知名角色的轮廓与配色。

## 2. 视觉信息优先级

注意力从强到弱必须是：

1. 小猫轮廓与当前运动状态。
2. 当前平台边缘、下一平台顶面和中间空隙。
3. 危险物和可交互 UI。
4. 近景屋顶装饰。
5. 中远景城市、霓虹和粒子。

背景、招牌、蒸汽或粒子只要与平台顶面竞争，就必须降低亮度、饱和度、对比度或运动幅度。装饰不能伪装成可落平台。

## 3. 像素与坐标系统

- 手机逻辑视口为 `390 × 844`；桌面逻辑高度仍为 844，横向可扩展到约 1055。
- 世界素材以逻辑像素绘制，主网格为 2px，建筑模块优先使用 8px 或 16px 倍数。
- 精灵最终绘制坐标取整；关闭图像平滑；禁止单个精灵非等比缩放。
- 小猫动画单元格为 `32 × 32`。可见轮廓约 `26 × 28`，物理碰撞盒保持 `24 × 28`。
- 小猫锚点位于碰撞盒底边中央。耳朵、尾巴和围巾可以超出碰撞盒，但不能影响判定预期。
- 平台顶边至少 6px 高，并使用贯穿整个平台的连续明亮轮廓。
- 像素线条不使用半透明抗锯齿；半透明仅用于大块雾气、远景和遮罩。

## 4. 核心调色板

首版控制在约 18 个核心色内；单个角色或场景模块通常不超过 6–8 色。

| 色彩角色 | 色值 | 用途 |
| --- | --- | --- |
| Ink 950 | `#0C0B1C` | 页面最深背景、轮廓阴影 |
| Night 900 | `#17162F` | 主夜空、面板底色 |
| Night 800 | `#211A3B` | 建筑暗面 |
| Violet 700 | `#312856` | 远景建筑、暗紫层次 |
| Violet 600 | `#493252` | 近景砖墙主体 |
| Haze 500 | `#6B5480` | 雾气、低对比轮廓 |
| Text Cream | `#FFF5D6` | 主文字、关键高光 |
| Cat Cream | `#FFF0C4` | 小猫口鼻、胸口和爪 |
| Warm Yellow | `#FFCF71` | 主按钮、平台提示高光 |
| Cat Orange | `#F6C26B` | 小猫主色 |
| Roof Orange | `#F0A45E` | 平台顶边、砖块暖光 |
| Window Amber | `#D89061` | 窗灯和环境暖点 |
| Scarf Coral | `#DF665C` | 围巾、品牌记忆点 |
| Neon Cyan | `#72D7D0` | 少量未来感招牌 |
| Neon Pink | `#E985B5` | 少量装饰霓虹 |
| Danger Red | `#FF5D6C` | 危险物，必须搭配形状 |
| Smoke Light | `#B8AECB` | 蒸汽高光、次要文字 |

使用规则：

- 暖黄表示“可落、可按、可亲近”，不把大面积背景染成暖黄。
- 危险红不能单独传达危险，必须配合尖角、闪烁节奏或警告图标。
- 青色和粉色霓虹合计不超过单屏视觉面积约 8%。
- 主文字使用 Text Cream，次要文字使用 Smoke Light。
- 新颜色必须说明现有调色板为何无法满足，禁止每个素材自行创造近似色。

## 5. 主角设计

当前选定方向：以概念板第一版的圆润豆形比例为品牌基础，吸收第二版侧身奔跑时较清楚的四肢与围巾方向；尾巴保持中等尺寸，不采用第三版会压过身体轮廓的大尾巴。概念板见 [M3 方向记录](M3_CONCEPT_NOTES.md)。

### 轮廓

- 奶油橘猫，头身比例约 1:1.1，耳朵短而清楚，身体略呈豆形。
- 红珊瑚色小围巾是远距离识别点，尾端朝运动反方向摆动。
- 尾巴表达平衡和速度，但不能与平台边缘混成一条线。
- 眼睛采用 2–3px 深色块，不加入正常尺寸下无法辨识的细节。

### 动画表

| 状态 | 建议帧数 | 播放速度 | 重点 |
| --- | ---: | ---: | --- |
| ready/idle | 4 | 4 fps | 呼吸、耳朵轻动、围巾轻摆 |
| run | 6 | 10–12 fps | 身体上下不超过 2px，脚步清楚 |
| jump-up | 2 | 8 fps 或按物理阶段切换 | 身体拉长、前爪收起 |
| apex | 1 | 保持 | 轮廓最圆，便于感知最高点 |
| fall | 2 | 8 fps | 身体略压缩、尾巴上扬 |
| land | 3 | 总时长 100–140ms | 轻微压缩后恢复，不锁输入 |
| fail | 4 | 8 fps | 惊讶而非受伤，避免暴力 |

动画只表达状态，不改变物理。落地动画即使未播放完，下一次合法跳跃也必须响应。

实现记录（2026-07-18）：首版运行时采用代码原生 32×32 像素网格，不加载外部精灵图。上述帧数已全部接入；落地动画总长 132ms，脚底以 28px 碰撞盒底边为锚点。`prefers-reduced-motion` 下所有循环动画冻结在第 0 帧，并跳过落地压缩与尘粒。若后续素材复杂度需要 PNG 精灵表，再按同一锚点和帧率导出，不改变物理层。

## 6. 屋顶与可玩场景

### 可落平台

- 顶面使用连续 Roof Orange 或 Warm Yellow 高光，暗部使用 Violet 600。
- 平台顶面附近 12px 内避免高亮窗户、霓虹横线和类似平台的远景边缘。
- 平台宽度变化通过模块裁切实现，不拉伸砖块纹理。
- 边缘可加 2–4px 磨损，但不能让真实碰撞边界含糊。

### 模块库

- 16×16 砖墙：完整砖、半砖、暗砖、暖光砖。
- 8px/16px 屋檐和排水槽。
- 水箱、短天线、通风口、晾衣绳、花盆与纸箱。
- 蒸汽口只放在不会遮挡起跳边缘的位置。
- 装饰物默认不可碰撞；危险物必须有独立形状语言和碰撞说明。

实现记录（2026-07-18）：首批运行时模块采用代码原生像素绘制，包含屋檐边帽、通风口、短天线与水箱。每个平台最多一个装饰；起始平台与宽度小于 126px 的平台不放置装饰，水箱仅用于至少 196px 的宽平台。所有装饰离左右起跳/落地边缘至少 36px，选择与位置只依赖平台 ID 和宽度，不参与碰撞。通风口蒸汽使用低对比短像素块，`prefers-reduced-motion` 下完全关闭。

### 危险物

MVP 如加入危险物，首批只使用一种，例如漏电天线。它必须具备：

- 独立红色或粉色信号色。
- 向上或外扩的尖角轮廓，与圆润小猫和水平平台区分。
- 至少一种非颜色提示：周期闪烁、火花或警告纹样。
- 视觉边界与碰撞盒误差不超过约 2px。

## 7. 背景层与宽屏构图

| 层级 | 相机相对速度 | 对比度 | 内容 |
| --- | ---: | --- | --- |
| 天空 | 0 | 最低 | 深紫渐变、少量星点 |
| 远景 | 0.03–0.06 | 很低 | 天际线、远处电车、塔楼 |
| 中景 | 0.08–0.15 | 低 | 建筑、窗灯、克制霓虹 |
| 可玩层 | 1.0 | 最高 | 平台、小猫、危险物 |
| 前景 | 1.05–1.15 | 中低 | 少量管线和雾气，不遮落点 |

移动端：

- 小猫通常位于屏幕宽度约 28%–32%、高度约 65%–75%。
- 保证下一平台入口可见，UI 不压住起跳与落地区域。
- 一屏通常不超过一个显眼霓虹招牌。

桌面宽屏：

- 小猫尺寸和物理单位保持不变，不能为了填满屏幕而放大角色。
- 额外横向空间展示 3–5 段屋顶、更多天际线和环境叙事。
- 远景可平铺到至少 1100 逻辑像素，不能在手机宽度之外露出空白或重复接缝。
- 右侧远方平台可以略暗，但下一落点必须与手机端同样清楚。
- 开始、暂停和失败面板保持可读最大宽度，不随游戏区无限放大。

## 8. UI 与字体

- 标题和分数可以使用许可明确的像素字体；正文、按钮和设置使用高可读无衬线字体。
- 中文像素字体未确定前，不把临时字体烘焙进图片。
- 主按钮使用暖黄底、深紫文字和清楚的按下位移；触控区域至少 44 CSS px。
- 分数使用等宽数字，至少容纳四位，增加位数时不推动暂停按钮。
- 面板优先深紫不透明底；模糊效果关闭后仍须可读。
- 键盘焦点使用高对比外圈，不能只改变颜色。

## 9. 动效与减少动态

- 正常模式允许围巾摆动、蒸汽、星点、落地尘粒和一次轻微镜头反馈。
- 屏幕震动最大 2–3 逻辑像素、持续不超过 100ms；普通跳跃不震屏。
- 同屏粒子设上限，粒子不能形成与平台顶边相似的长横线。
- `prefers-reduced-motion` 下关闭震屏、快速闪烁、大幅视差和多数粒子。
- 失败反馈不能依赖闪白，避免高频闪烁。

## 10. 制作与导出

- 可编辑源文件与导出文件分开保存，源文件保留图层、帧和调色板。
- PNG 使用透明背景、无插值缩放；禁止有损压缩精灵图。
- 文件名使用小写 kebab-case，例如 `cat-run.png`、`roof-brick-a.png`。
- 精灵表从左到右排列，元数据记录帧数、帧率、锚点和碰撞盒。
- 每项素材在 `docs/ASSET_MANIFEST.md` 记录作者、来源、许可和状态。
- 第三方参考图只用于讨论气氛，不能进入项目包，也不能逐像素临摹。

## 11. 制作顺序

1. 调色板小样：同屏验证猫、平台、危险信号和背景层级。
2. 小猫轮廓表：比较至少三种纯色剪影，判断 24–32px 下的可辨识度。
3. 跑、跳、落地动画灰阶版：先验证动作，再上色。
4. 一套完整屋顶模块：验证随机宽度拼接和平台边缘连续性。
5. 一屏远中近背景：同时检查 390 与约 1055 逻辑宽度。
6. UI 和字体：统一标题、分数、按钮与面板。
7. 视觉回归和性能压缩：通过后再批量制作变体。

不要先制作大量装饰或多套皮肤。第 1–5 步形成一屏垂直切片并通过验收后，再扩展素材数量。

## 12. M3 视觉验收门槛

- 360px 宽手机截图中，小猫、平台顶边和空隙无需放大即可辨认。
- 截图转为灰度后，主角和下一平台仍明显高于背景层级。
- 390 与桌面宽屏使用同一素材尺度，不存在拉伸、接缝或空白。
- 所有动画状态与物理一致，动画不会吞掉输入。
- 危险物不只依赖红色，视觉边界与碰撞基本一致。
- 减少动态模式无震屏和快速闪烁，玩法信息不丢失。
- 所有素材都有作者、来源和许可，没有权属不明文件。
- 连续三局没有纹理抖动、素材加载闪烁或明显性能恶化。

自动化灰度门槛：Cat Orange 对 Night 的对比度必须大于 7:1，Roof Orange 对 Violet 600 必须大于 4.5:1；远景 Violet 700 对 Night 必须小于 1.5:1。该门槛用于防止后续调色让背景抢过主角或落点。
