# M1/M2 校准与试玩记录

更新日期：2026-07-18

## 当前手感基线

- 模拟频率：固定 120 Hz；渲染继续使用 `requestAnimationFrame`。
- 初始跑速：150 逻辑像素/秒；中段 162；高分段 175。
- 跳跃初速度：-370 逻辑像素/秒；重力 1050 逻辑像素/秒²。
- 同高度完整滞空时间约 0.705 秒。
- 计分：每前进 15 逻辑像素得 1 分，每首次落到一个新平台奖励 8 分。

| 分数 | 跑速 | 平台间隙 | 高度变化 | 平台宽度 | 相机最大追踪速度 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 0–149 | 150 | 54–68 | -12–12 | 170–230 | 14 |
| 150–499 | 162 | 62–80 | -22–28 | 130–190 | 18 |
| 500+ | 175 | 68–90 | -30–40 | 105–160 | 23 |

初级同高度平台在最大间隙下仍保留约 0.25 秒的有效起跳窗口；较短间隙约为 0.34 秒。生成器同时检查跳跃可达性和直接走下平台的坠落轨迹，确保每个间隙需要主动跳跃。

## 响应式视野

- 手机保持 `390 × 844` 逻辑基准，CSS 只做等比缩放。
- 使用精细指针且视口至少 768×600 时，桌面游戏区扩展为约 `1.25:1`。
- 1440×900 浏览器中实测游戏区约为 1055×844，Canvas 逻辑宽度约 1057；画面不是将 390px 手机视图横向拉伸。
- 宽屏只增加可见世界和预生成范围，不改变物理、平台序列或计分规则。

## 已完成验证

- 300 个随机种子 × 每个 80 个平台：全部可达，且不能不跳直接走落到下一平台。
- 固定种子理想输入连续模拟 180 秒：未失败，跨越 0–150、150–500、500+ 三阶段。
- 浏览器键盘路径：开始、连续跳跃、失败、重开、暂停和恢复正常。
- 指针路径：390×844 视口连续点击跳跃正常。
- 视口：360×640、390×844、430×932、1440×900 无页面溢出。
- 浏览器运行期间无控制台警告或错误。
- 修复长局面垂直相机漂移：相机只朝已安全落地的平台高度平滑追踪；50 个种子各运行 60 秒和固定种子 180 秒均保持可玩纵向构图。

## 物理手机待验清单

自动化视口不能替代真实触屏、地址栏伸缩和设备性能测试。发布前至少完成：

完整步骤、通过门槛和报告模板见 [真实手机验收手册](PHONE_ACCEPTANCE.md)。

当前设备结果见 [M2 实机验收记录](M2_ACCEPTANCE_REPORT.md)。

1. iPhone Safari 与 Android Chrome 各连续玩三局，每局记录时长、分数和误触。
2. 快速连点、两指同时触摸、从屏幕边缘触摸，确认每次只产生一个合法跳跃。
3. 收起/展开浏览器地址栏，确认游戏区不跳动、不被底部手势区遮挡。
4. 切到后台 10 秒再返回，确认停在暂停页且必须显式继续。
5. 连玩至少 3 分钟，观察发热、掉帧、音画延迟和内存增长。
6. 横屏进入后看到竖屏提示；恢复竖屏后状态正常。

同一 Wi-Fi 下可在开发机运行：

```bash
npm run dev -- --host 0.0.0.0
```

然后在手机访问终端输出的 `Network` 地址。该地址会随网络变化，不要写入应用代码或公开文档链接。
