## 2. 音效、静音与输入隔离

- 使用原生 Web Audio 振荡器生成起跳、落地、得分和失败短音效，不下载外部音频。
- 音频只在用户手势后创建/恢复；创建或播放失败时静默降级，不影响开始和游戏循环。
- 静音按钮为 44×44 CSS px，具有明确的 `aria-label` 与 `aria-pressed` 状态。
- 静音状态写入 schema v2；schema v1 的最高分迁移后保留，声音默认开启。
- 点击静音按钮时分数保持 `0000`、开始遮罩仍显示，证明 UI 点击没有误触跳跃。
- 静音后刷新仍显示“开启声音”；重新开启后开始游戏，计分和暂停按钮正常。
- 浏览器验收发现并修复：开始/失败遮罩曾覆盖 HUD 点击区域；HUD 层级现已高于遮罩，横屏提示仍最高。

## 5. 动画与灰度专项（2026-07-18）

- 运行时正式采用代码原生像素帧：idle 4、run 6、jump-up 2、apex 1、fall 2、land 3、fail 4。
- 落地动画固定 132ms，只读取物理状态，不锁输入、不改碰撞；检测到落地时同步触发轻量尘粒。
- 浏览器局部截图发现首版落地脚底曾比碰撞盒多探出约 3px，已将身体压缩改为向下收短，并把脚底重新锚定到平台顶边。
- 360×640 上升帧仍有清楚的向右侧身轮廓；390×844 落地帧与平台亮边对齐；430×932 和 1440×900 无布局回归。
- 桌面游戏区仍为 1055×844 CSS px，页面无意外滚动；浏览器控制台无 warning/error。
- `prefers-reduced-motion` 下循环帧固定为第 0 帧，跳过落地压缩、尘粒、星点和远中景视差；不改变物理与计分。

## 6. 屋顶模块专项（2026-07-18）

- 新增确定性的屋檐边帽、通风口、短天线、水箱和通风口蒸汽；选择、位置和蒸汽有无只由平台 ID/宽度决定，不消耗玩法随机数。
- 起始平台和宽度小于 126px 的平台不放装饰；天线至少需要 150px，水箱至少需要 196px。每个平台最多一个装饰，左右各保留至少 36px 的清楚边缘。
- 装饰先于平台主体绘制、猫后绘制，底座自然藏入屋顶且猫经过时保持前景层级；装饰完全不参与碰撞。
- 390×844 自动连续跳跃到 66 分仍可辨识平台空隙，暂停后正确显示“稍作休息”；360×640、430×932 均无滚动或 HUD 挤压。
- 1440×900 游戏区为 1055×844 CSS px，额外视野中的装饰密度克制，猫和物理单位无拉伸。
- 水箱和天线使用 Haze/Violet Dark 等低对比色，不形成暖色可落长横线；蒸汽只使用短像素块，减少动态模式完全关闭。
- 四视口浏览器回归控制台均无 warning/error。

## 8. 环境音与连续运行专项（2026-07-18）

- 环境音使用两个低音载波、一个低频调制器和低通滤波器生成，主增益为 `0.0032`；无下载音频、无随机采样、首包只增加代码。
- 环境音只在 `playing` 且未静音时存在；暂停、页面中断、失败和静音会停止并断开旧节点，继续、重开或重新开启声音时最多创建一套固定大小的图。
- 音频控制器测试覆盖静音不创建上下文、重复启用幂等、暂停/静音释放、创建失败降级，以及连续 50 局启停后所有旧环境声源均已停止。
- 同一个 `GameSession` 连续重置并运行 3 个各 180 秒的理想输入长局，三局均超过 1500 分且未失败；宽屏模式下运行时平台集合始终不超过 24 个，证明已生成世界会被持续裁剪。
- 本地浏览器在 120Hz 桌面环境进行了两轮连续三局采样，每轮 10 秒并包含环境音：两轮平均帧分别为 8.33ms、8.32ms，最慢帧均为 10.4ms，超过 25ms/50ms 的帧均为 0。
- 第一轮包含页面启动与即时编译，JS 堆变化约 `+1.16 MB`；热身后第二轮约 `-2.23 MB`，未观察到逐轮单调增长。采样探针测试后已从源码完全移除。
- 浏览器交互确认开始、暂停、继续、游玩中静音、失败与重开均正常；声音按钮的 `aria-pressed` 与标签同步，UI 操作没有误触跳跃。
- 上述数据是桌面工程证据，不能冒充低性能真机结果；最终 iPhone 复验步骤已写入 `PHONE_ACCEPTANCE.md` 第 8 节。
