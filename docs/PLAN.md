# 赛博小猫跳楼顶：项目计划书

## 1. 项目概览

**暂定名称**：赛博小猫跳楼顶 / *Cyber Cat Rooftops*  
**类型**：单人、竖屏、像素风、轻度跑酷网页游戏  
**目标用户**：通过链接偶然进入或由朋友分享而来的公开用户；优先服务手机用户，同时保证电脑端有完整体验。  
**核心承诺**：无需下载、无需登录、三秒内开始游戏、一次操作就能理解。

玩家控制一只可爱的小猫，在向下推进的城市楼顶之间自动奔跑与跳跃。镜头随已落地楼顶的高度向下追踪，楼顶、招牌、热气与夜景从前方出现。随着得分增长，相机追踪和跑动节奏加快、平台间距更有变化，玩家需要更快判断落点。落空即结束，可立即再来一局。

### 成功标准（首版）

1. 首屏加载完成后，用户在 3 秒内能看到并进行第一次跳跃。
2. 初次玩家不看说明也能理解目标；提示只在第一次出现。
3. 在 iPhone 尺寸、安卓常见尺寸、桌面键盘和鼠标下，操作无明显误触或延迟。
4. 60 秒内能完成完整的“开始 → 挑战 → 失败 → 重开”循环。
5. 部署到 RackNerd VPS 并经 Cloudflare 代理后，静态游戏不依赖应用服务器。

## 2. 首发版本范围（MVP）

### 包含

- 手机端使用竖向游戏画面，逻辑基准尺寸为 `390 × 844`；桌面端扩展横向视野。
- 自动向右奔跑的小猫；玩家仅控制跳跃。
- 稳定的重力、跳跃、落地、边缘跌落和碰撞判定。
- 程序生成的楼顶平台：不同宽度、高度、装饰与安全间距。
- 追踪已落地楼顶高度的镜头；相机追踪速度会逐级提升，但不得脱离地形自行滚动。
- 分数、当前最高分、本地存档（`localStorage`）。
- 开始、暂停、失败、重玩及首次操作提示。
- 桌面与触控输入支持；音效开关；减少动态效果偏好支持。
- 原创温暖像素风小猫、楼顶、远景与 UI 素材。
- 基础分享文案和 Open Graph 预览图。

### 首版不包含

- 登录、好友、实时对战、公会、聊天、付费道具。
- 多角色、复杂剧情、关卡编辑器、广告系统。
- 全球排行榜（作为第二阶段，避免过早处理作弊和数据治理）。

## 3. 玩法与数值设计

### 操作

| 设备 | 跳跃操作 | 补充 |
| --- | --- | --- |
| 键盘 | `Space`、`↑`、`W` | 按键仅在游戏区域激活时生效。 |
| 鼠标 | 左键点击游戏画面 | 点击菜单、暂停与设置不触发跳跃。 |
| 触屏 | 轻触游戏画面 | 只处理第一根手指，禁用页面双击缩放干扰。 |

首版采用**单次跳跃**，不引入二段跳，保留最清晰的风险判断。若测试显示前期挫败感过强，可在后续实验“短按低跳、长按高跳”，但不应阻塞 MVP。

### 游戏循环

1. 玩家进入落地状态，点击或按键启动。
2. 小猫自动横向跑动；玩家在楼顶边缘前跳跃。
3. 落到下一座楼顶继续累计分数；镜头下移带来新楼顶。
4. 漏跳、跳过平台或撞到明确的危险物时结束。
5. 显示本局分数、本地最高分和“再来一局”。

### 难度曲线（初始假设，需试玩校准）

- **0–150 分**：较宽平台、固定节奏、无强制障碍，帮助建立手感。
- **150–500 分**：平台宽度和间距开始波动，滚动速度每 100 分小幅增加。
- **500 分以上**：速度稳定加快，出现窄屋檐、霓虹招牌等视觉干扰；仍保证可通过的落点。
- 分数采用“前进距离 + 成功落地奖励”，以鼓励连续稳定发挥。
- 随机生成必须遵守“可达性”规则：每一平台至少存在一条符合小猫跳跃速度、滞空时间和水平移动范围的落点路径。

## 4. 视觉与声音方向

### 画风

温暖、可爱、清晰的像素风：深蓝紫夜空与暖橘窗光形成对比；霓虹只作点缀，不走冷硬的赛博朋克路线。主角轮廓必须在小尺寸屏幕上清楚可辨。

- **主角**：圆润橘白或奶油色小猫，佩戴小围巾/小背包作为轮廓记忆点。
- **场景**：砖墙楼顶、水箱、天线、窗户、晾衣绳、蒸汽口、远处电车与招牌。
- **分层**：远景天际线慢速移动，中景建筑适速移动，近景屋顶随游戏推进；产生纵深但不影响判读。
- **UI**：像素字体只用于标题和分数；按钮保持现代、足够大的点击区域以保障可读性。

### 音效

首版只需要跳跃、落地、得分、失败和极轻的环境循环音。默认开启但保存用户偏好；无法自动播放背景音乐。提供静音按钮与 `prefers-reduced-motion` 降低屏幕抖动/粒子效果。

## 5. 技术方案

### 客户端

- **框架**：Vite + TypeScript，游戏层优先使用 HTML5 Canvas 2D；页面 UI 用原生 DOM/CSS。
- **原因**：Canvas 可控、体积小，适合像素画面和固定逻辑分辨率；不为一个单场景小游戏引入大型引擎依赖。
- **渲染**：游戏以固定内部尺寸绘制，CSS 等比缩放；开启最近邻像素缩放（不拉伸、不模糊）。
- **循环**：`requestAnimationFrame` + delta time；标签页恢复时限制最大 delta，避免物理跳帧。
- **状态机**：`ready` → `playing` → `paused` → `gameOver`，所有输入按状态过滤。
- **数据**：本地最高分和设置保存于 `localStorage`，附带版本号和损坏数据兜底。

### 尺寸与设备适配

- 移动端：游戏尽可能填满可视高度，尊重安全区（刘海、底部手势区），锁定竖向体验提示但不强制系统方向锁。
- 桌面端：保持 844 逻辑高度并将横向视野扩展到约 `1.25:1`，最大 CSS 宽度约 1100px；不得把手机画面横向拉伸。
- 所有交互元素满足至少约 44px 的触控目标；画布不选中文本、不触发拖动或双击缩放。
- 键盘焦点丢失、窗口失焦和页面隐藏时自动暂停；恢复时要求用户显式继续，避免意外失败。

### 云端与部署

第一阶段使用 **RackNerd VPS + Caddy** 发布纯静态构建产物，并由 Cloudflare 管理 DNS、代理与边缘缓存。生产地址为 `https://game.norliva.top/`。游戏运行和物理计算发生在玩家设备；VPS 只提供静态文件，不承载游戏状态或用户数据。发布采用版本目录和原子软链接，保留最近 5 个版本用于快速回滚。

第二阶段如需全球排行榜，增加：

- **Cloudflare Workers**：`POST /api/scores`、`GET /api/leaderboard`。
- **Cloudflare D1**：仅保存昵称、分数、提交时间、版本及校验字段。
- **Cloudflare Turnstile**：减少机器人刷提交。
- **防作弊底线**：服务端速率限制、分数上限与增长合理性校验、会话签名/挑战令牌；排行榜说明中明确其为轻度休闲榜，不承诺竞技级反作弊。

不保存不必要的个人信息；若加入昵称，限制长度、过滤控制字符和敏感内容，并提供清晰的榜单说明。

## 8. 测试清单

### 功能

- 开始、暂停、恢复、失败、重开都不会叠加多个游戏循环。
- 角色不会穿透平台、边缘判定一致、镜头滚动不会吞掉可见平台。
- 同一随机种子下的场景可复现，便于调试。
- 本地最高分只在有效新记录时更新，清除存储后有正常默认值。

### 设备与性能

- 360×640、390×844、430×932 等手机视口；横屏时给出简洁提示。
- 1366×768、1440×900、4K 桌面；高 DPI 下像素边缘仍清晰。
- 低性能手机连续游玩三局后不明显掉帧或内存增长。
- 快速连点、多点触控、键盘长按、切后台后回来均不导致异常状态。

### 发布

- Lighthouse / Web Vitals 检查；压缩音频与图片，避免首包过大。
- 确认缓存头、页面标题、描述、favicon、Open Graph 图和分享链接。
- 记录 Cloudflare 项目名、生产域名、回滚方式和依赖版本。

## 9. 风险与处理

| 风险 | 处理方式 |
| --- | --- |
| 随机平台出现无法跳过的死局 | 用跳跃参数计算可达区间；保留随机种子重放。 |
| 手机网页手势影响游戏 | 在游戏区设置触控行为，拦截非 UI 区的默认手势；充分实机测试。 |
| 像素素材影响加载 | 精灵图、压缩与延迟加载；首屏只加载必要资源。 |
| 早期排名被刷 | 榜单延后上线；上线时加入 Turnstile、限流和合理性校验。 |
| 风格与既有作品混淆 | 使用原创名称、角色、玩法包装、素材和文案，不借用小恐龙资产或品牌。 |
